# 旅行規劃師 - 雲端協作版 ✈️

> 一個功能完整的多人協作旅行規劃系統，讓你和朋友輕鬆規劃每一趟旅程！

## 📋 專案簡介

**旅行規劃師**是一個基於雲端的群組旅行管理系統，專為多人旅行設計。無論是家族旅遊、朋友出遊或公司團建，都能透過這個系統輕鬆管理行程、分帳、打包清單等所有旅行細節。

### ✨ 核心特色

- 🌐 **完全雲端化** - 使用 Firebase 即時同步，所有成員隨時看到最新資訊
- 👥 **多人協作** - 支援群組管理，每個成員都能參與編輯
- 📱 **響應式設計** - 完美支援手機、平板、電腦各種裝置
- 🎯 **零安裝** - 純網頁應用，開啟瀏覽器即可使用
- 🔒 **帳號系統** - 獨立的用戶註冊登入，保護個人資料

## 🎮 主要功能

### 1. 📅 行程規劃
- **時間軸式呈現** - 以日期分頁顯示，清楚看到每天的安排
- **分類管理** - 交通、住宿、景點、餐飲、購物等類別
- **Google 地圖整合** - 每個行程點都能連結 Google 地圖
- **官網連結** - 可附加景點或餐廳的官方網站
- **備註功能** - 記錄注意事項、營業時間等重要資訊

### 2. ✅ 待辦清單
- **任務分配** - 指定負責人，明確分工
- **分類標籤** - 依類型整理待辦事項
- **完成追蹤** - 即時更新完成狀態
- **群組共享** - 所有成員都能查看和編輯

### 3. 🎒 整理行李
- **個人清單** - 每個人管理自己的打包物品
- **打包進度** - 勾選已打包項目，避免遺漏
- **快速新增** - 簡單輸入即可添加物品

### 4. 💰 個人記帳
- **私人帳本** - 記錄個人支出，不與他人共享
- **分類統計** - 自動計算各類別花費
- **明細查詢** - 完整的支出歷史記錄
- **總額顯示** - 即時顯示總支出

### 5. 💳 群組分帳
- **智能分帳** - 自動計算每個人應付金額
- **參與者選擇** - 彈性選擇分攤成員
- **結算建議** - 自動產生最少轉帳次數的結算方案
- **餘額追蹤** - 即時顯示每個人的應收/應付款

### 6. 💱 匯率換算
- **即時匯率** - 串接 API 取得最新匯率
- **多幣別支援** - 支援 16 種常用貨幣
- **雙向換算** - 可互相換算任意幣別
- **快速參考** - 一目了然的匯率總覽

### 7. 💬 群組聊天
- **即時通訊** - 群組成員即時溝通
- **訊息記錄** - 保留完整對話歷史
- **自動刷新** - 每 5 秒自動更新訊息
- **時間戳記** - 清楚顯示訊息時間

### 8. 🔧 系統管理 (管理員專用)
- **用戶管理** - 查看所有註冊用戶
- **群組管理** - 管理群組成員、刪除群組
- **成員調配** - 將用戶加入或移出群組
- **資料維護** - 系統資料檢視與維護

## 🛠️ 技術架構

### 核心技術
- **單一 HTML 檔案** - 所有程式碼整合在一個檔案中，方便部署
- **原生 JavaScript** - 使用 Firebase SDK 進行資料管理
- **Tailwind CSS (CDN)** - 實用優先的 CSS 框架
- **響應式設計** - 自適應各種螢幕尺寸

### 雲端服務
- **Firebase Firestore** - NoSQL 雲端資料庫，即時同步
- **Firebase Analytics** - 使用分析

### API 整合
- **ExchangeRate API** - 即時匯率查詢
- **Google Maps** - 地圖導航功能

### 特色設計
- **CDN 載入** - 無需本地安裝套件，開箱即用
- **離線備援** - 匯率功能提供離線模式
- **自動登入** - 記住登入狀態，提升使用體驗

## 📦 安裝與部署

### 方法 1: 直接使用 (最簡單)
1. 下載 `index.html` 檔案
2. 用瀏覽器開啟即可使用

### 方法 2: 架設到網站伺服器
```bash
# 上傳到任何靜態網站託管服務
# 如: GitHub Pages, Netlify, Vercel, Firebase Hosting

# 以 Firebase Hosting 為例:
firebase init hosting
firebase deploy
```

### Firebase 設定

**重要：必須先設定 Firebase 才能正常使用！**

1. 前往 [Firebase Console](https://console.firebase.google.com/)
2. 建立新專案
3. 啟用 Firestore Database
4. 取得專案設定值（專案設定 > 一般 > 您的應用程式 > 網頁應用程式）
5. 替換 `index.html` 中的 `firebaseConfig` 設定：

```javascript
const firebaseConfig = {
    apiKey: "你的-API-KEY",
    authDomain: "你的-專案.firebaseapp.com",
    projectId: "你的-專案-ID",
    storageBucket: "你的-專案.firebasestorage.app",
    messagingSenderId: "你的-ID",
    appId: "你的-APP-ID",
    measurementId: "你的-測量-ID"
};
```

### Firestore 安全規則設定（建議）

在 Firebase Console 的 Firestore Database > 規則，設定以下規則：

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /travel_app/{document=**} {
      allow read, write: if true;  // 開發階段
      // 正式環境建議加上認證限制
    }
  }
}
```

## 🚀 使用指南

### 首次使用

1. **註冊帳號**
   - 輸入用戶名和密碼
   - 第一個註冊的用戶自動成為管理員

2. **建立群組** (管理員)
   - 點擊「建立新群組」
   - 輸入群組名稱（建議用旅行目的地）

3. **加入成員** (管理員)
   - 進入「系統管理」
   - 在用戶管理中將成員加入群組

4. **開始規劃**
   - 選擇群組進入
   - 開始使用各項功能規劃旅行

### 權限說明

- **管理員**
  - 建立/刪除群組
  - 管理群組成員
  - 查看系統統計
  - 使用所有一般功能

- **一般用戶**
  - 查看已加入的群組
  - 編輯群組內容
  - 管理個人資料

## 💡 使用技巧

### 行程規劃建議
1. 先建立每天的住宿和交通行程
2. 再填入景點和餐廳
3. 善用 Google 地圖連結規劃路線
4. 使用備註記錄營業時間、預約資訊等

### 分帳管理技巧
1. 隨時記錄群組花費
2. 標註清楚付款人和參與者
3. 定期查看結算建議
4. 旅行結束後一次結清

### 打包清單妙用
1. 出發前一週開始列清單
2. 參考以往旅行經驗
3. 打包時逐項勾選
4. 避免遺漏重要物品

### 匯率換算應用
1. 出發前先查詢匯率
2. 估算旅行預算
3. 購物時快速換算
4. 比較不同幣別價格

## 🔐 資料安全

- 所有資料儲存在 Firebase 雲端
- 支援 Firebase 安全規則設定
- 建議設定 Firestore 權限控管
- 密碼建議使用強密碼
- 定期備份重要資料

**隱私說明：**
- 個人記帳資料：僅自己可見
- 打包清單：僅自己可見
- 群組資料：群組內所有成員可見可編輯

## 🐛 已知問題與限制

1. **密碼安全性**：目前密碼為明文儲存，建議不要使用重要密碼
2. **並發編輯**：多人同時編輯可能有衝突（採用最後寫入優先）
3. **檔案上傳**：目前不支援上傳圖片或附件
4. **離線功能**：需要網路連線才能同步資料
